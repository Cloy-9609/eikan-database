# 非機能要件

## 性能
- 一覧画面は実用的な速度で表示されること
- レスポンス：1秒以内
- 同時接続：10人想定
- 画像はローカル保存を前提とする
- 画像パス切れ時はデフォルト画像またはプレースホルダを表示する

## 保守性
- フロントエンドとバックエンドを分離する
- ドキュメントを機能別に整理する
- 仕様変更時は `requirements`、`design`、`decisions` の関連資料を合わせて更新する  

エラー時:
- APIエラーはメッセージ表示
- 画像未取得時はデフォルト画像

## 可読性
- 命名規則を統一する
- レイヤー構造を維持する

## 拡張性
- 学校・選手以外のデータ種別を追加しやすい構造にする

## 運用前提
- ローカル利用または少人数利用を主な前提とする
- 画像ファイルと SQLite ファイルは定期的なバックアップを前提とする

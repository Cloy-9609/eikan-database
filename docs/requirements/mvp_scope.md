# MVP 定義

## この文書の扱い

この文書は Phase 1 開始時の MVP 範囲を定義した歴史資料である。
MVP は完了済みであり、現在の実装はこの MVP 範囲を超えている。

独立した `players.html`、`player_series` / snapshot 管理、検索・絞り込み、能力範囲検索、sort、守備位置図、学校年度進行、管理コード基盤などは Phase 2 で追加された成果である。
現在地は `docs/phases/phase2.md` を参照する。

## MVP の目的

最小構成で学校・選手データの登録、参照、更新を成立させる。

## 含める機能

### 学校
- 学校作成
- 学校一覧
- 学校詳細
- 学校編集
- 学校削除（論理削除）

### 選手
- 学校詳細内での所属選手一覧表示
- 選手登録（手動）
- 選手詳細
- 選手編集

## 含めない機能

- 高度な検索
- OCR 自動取り込み
- 権限管理
- 選手削除
- 独立した選手一覧ページ

## MVP 完了条件

- 学校 CRUD が画面と API の両方から利用できる。
- 学校詳細から所属選手を参照できる。
- 選手の登録、詳細表示、編集が成立している。
- Phase 1 時点の基盤として、`schema.sql` を中心にした初期データ構造と DB 初期化手順が確立している。

補足: 上記の `schema.sql` ベースの確立は Phase 1 の土台を意味する。将来一切 schema を変更しないという意味ではない。

## 検討項目

- Phase2 以降に OCR 補助入力の導入を検討する。

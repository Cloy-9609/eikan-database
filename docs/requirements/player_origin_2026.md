# 選手出身地の2026対応方針

## 目的

2026新作で海外からの選手が増えた場合でも、既存データと現行実装を壊さずに、選手系UIでは自然な「出身地」表現で扱えるようにする。

## 現行互換性

- 現行DB/APIでは、選手の出身地に相当する値は互換性維持のため `prefecture` フィールドを使う。
- 今回はDB schema変更、migration追加、APIフィールド名変更、保存・更新・削除処理の変更は行わない。
- 既存データの変換処理は実装しない。
- 内部フィールド名 `prefecture` は当面残し、UI表示だけを選手向けに読み替える。

## UI表示方針

- 選手系画面では、`prefecture` を「出身地」または必要に応じて「出身都道府県・国」と表示する。
- 日本の選手は都道府県、海外選手は国名として扱う。
- raw key や内部名としての `prefecture` は、通常のユーザー向け表示には出さない。
- 学校所在地と選手出身地はUI上の意味が異なるため混同しない。

## 学校所在地との区別

- 学校所在地は日本国内の都道府県として扱う。
- 学校系画面の「都道府県」文言は現状維持する。
- この方針は、選手の出身地表示を自然にするためのものであり、学校系画面の「都道府県」を「所在地」などへ置き換えるものではない。

## constants の扱い

- `backend/constants/prefectures.js` と `frontend/js/constants/prefectures.js` の構造は維持する。
- `PREFECTURE_GROUPS` と `ALLOWED_PREFECTURE_VALUES` は、既存の日本都道府県リストと海外国リストを扱うために継続利用する。
- 既存の「その他」は残す。
- 2026で追加国が判明した場合のみ、確認済みの国名を constants に追加する。
- 未確認の国を推測で追加しない。

## 将来検討

将来的には、以下のように出身地をより明示的に分離する可能性がある。

- `origin_region`
- `origin_country`
- `origin_type`

ただし、今回の対応ではこれらの新規フィールド追加やデータ移行は行わない。

# 削除ポリシー

## 基本方針
- 学校削除は原則として論理削除（`is_archived`）で扱う
- 理由：
  - 誤削除防止
  - 履歴保持

## 学校削除時の扱い
- `DELETE /api/schools/:id` は物理削除ではなく、`schools.is_archived = 1` への更新を意味する
- 削除済み学校は通常の学校一覧および学校詳細の対象外とする
- 配下の選手データは削除しない
- 配下の選手は通常の一覧 API / 一覧画面からは非表示にする
- 配下の選手は `GET /api/players/:id` と選手詳細画面でのみ参照可能とし、将来機能で再利用できる状態を保つ
- 削除済み学校に対する新規選手登録と既存選手更新は拒否する

## 復元機能
- データ上は `is_archived = 0` に戻すことで復元可能
- 現時点では復元専用の API / UI は未実装

## 物理削除
- 将来の管理者向け機能として検討する
- 現時点では完全削除 API は未実装

## 検討項目
- 削除済み学校・削除済み学校所属選手の専用一覧 UI
- 復元時に学校配下選手をどう再表示するか
- 監査ログの要否
- 管理者制御は将来機能（現状は未実装）

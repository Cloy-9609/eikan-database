# OCR ポリシー

## 現時点の方針
- MVP では OCR を必須機能に含めない

### 入力
- 画像（png/jpg）

### 出力（候補値 JSON）
{
  "name": "",
  "meat": 50,
  "power": 60,
  ...
}

### フロー
画像 → OCR → 候補値 JSON → UI 上で確認・修正 → API送信

### 失敗時
- 手動入力にフォールバック

## 将来対応
- 手入力を優先し、必要に応じて OCR 補助機能を検討する
- OCR 出力は「候補値」として扱う
- UI 上でユーザー確認後に登録する

## 検討観点
- 認識精度
- 修正フロー
- コスト

# 技術スタック

## フロントエンド
- HTML
- CSS
- JavaScript（ES Modules）

## バックエンド
- Node.js
- Express

## データベース
- SQLite
- `sqlite3` パッケージ

## 技術選定理由

### フロントエンド
- 画面数がまだ少なく、素の HTML / CSS / JavaScript で十分に構築できる。
- MVP 段階では依存ライブラリを増やさず、画面仕様の確定を優先できる。

### Node.js / Express
- API サーバーを小さく素早く構築しやすい。
- ルーティング、サービス、モデルの責務分離がしやすく、MVP の構成と相性がよい。

### SQLite
- ローカルファイルベースで運用でき、個人開発や検証用途に向いている。
- 現行実装が `backend/db/schema.sql` と `backend/db/database.js` を通じて SQLite 前提で構成されている。
- サーバー構築コストを抑えたまま、リレーショナルな制約や正規化を扱える。

## 判断メモ
- MVP はシンプルな構成を優先する。
- データ量や同時利用者が増えた場合は、将来的に別 RDBMS への移行を再検討する。

# Admission Year Picker 回帰確認・仕様書追加レポート

## 1. 背景

- `player_register.html` に実装した西暦チェッカーの挙動について、実ブラウザで本家パワプロ相当の動きが確認された。
- 続いて `player_edit.html` 側でも同様の確認が行われ、問題なしと判断された。
- あわせて、今後の参照用として入学年ピッカーの最終仕様書を `docs/requirements/` に追加することになった。

## 3. 実施した確認

### 3.1 実ブラウザ確認

- `schools.html` の起動を確認。
- 学校データ消失後、`POST /api/schools` によりテスト学校を再作成。
- `school_detail.html` から `player_register.html` へ遷移し、西暦チェッカーの挙動確認を実施。
- `player_edit.html` でも同様の確認を実施。
- 登録画面、編集画面ともに問題なしと判断された。

### 3.2 読み取り系回帰確認

- SQLite 上の有効データを確認した。
- 有効学校 1 件、選手 1 件が存在することを確認した。
- `startServer()` による起動が成功することを確認した。
- 以下の API / ページ取得が成功することを確認した。
  - `GET /api/schools`
  - `GET /api/schools/1`
  - `GET /api/players/1`
  - `GET /pages/player_register.html?school_id=1`
  - `GET /pages/player_edit.html?id=1`

### 3.3 入学年ピッカーロジック確認

- `isValidYear()` の境界値確認
  - `1932` は有効
  - `1931` は無効
  - `2039` は有効
  - `2040` は無効
- `getSafeAdmissionYear()` の丸め確認
  - 下限未満は `1932`
  - 上限超過は `2039`
- `getNextAdmissionYear()` の代表ケース確認
  - 千の位: `1932 -> 2039`, `2039 -> 1932`
  - 百の位: `1932 -> 2032`, `2000 -> 1932`
  - 十の位: `1940 -> 1939`, `2039 -> 2009`
  - 一の位: `1932 -> 1933`, `2039 -> 2030`
- `buildAdmissionYearPicker()` の出力に hidden input `admission_year` が含まれることを確認した。

## 4. 確認結果

- ユーザーによる実ブラウザ確認では、登録画面・編集画面ともに期待どおり動作した。
- 読み取り系 API と対象 HTML は取得可能であり、画面遷移の前提は維持されている。
- 入学年ピッカーの境界値、初期値補正、桁別遷移は現行実装どおりに動作している。
- 回帰確認の範囲では、新たな不具合は確認されなかった。
